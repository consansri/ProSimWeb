Code Editor

Controls

  • switches to transcript view
    click on horizontal name to switch between compiled and disassembled transcript
  • shows the compilation state of the current file
    until 250 lines code the editor will queue a compilation event within a few seconds
  • undo current changes in current file
  • redo current changes in current file
  • shows code editor info and shortcuts on hover
  • build the current file
  • cleares current file content
  • shows grammartree information of selected element in code for debugging

Execution Control

  • continuous program execution (limit at 1000 instructions)
  • executes next instruction
  • executes certain amount of instructions
  • executes the upcoming subroutine (behaves as a single step if the next instruction is not a jump)
  • return subroutine until a certain return instruction
  • resets memory, reassembles (from already existing syntax tree) current program, resets registers and pc

Registers

  • access multiple register files by switching between the tabs right beside "Register"
  • switch binary interpretation between BIN (binary), HEX (hexadecimal), UDEC (unsigned decimal) and DEC (signed decimal)
    by click at the type name in the table head
  • edit values
  • names and aliases both can be used as a register reference in code editor
  • switch between reg view with descriptions or 2 register views besides each other

Memory

  • opens the edit window where you can specify the start and width of memory mapped io (editable directly in memory)
  • change between Little Endian and Big Endian
    recompilation will be necessary before execution!
  • change amount of memory instances which should be shown in one row instead of directly remapping the instances on each change
  • reverse row address sorting (big to little, little to big)
